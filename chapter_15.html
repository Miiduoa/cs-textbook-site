English Reading Guide (中文導讀)

資料庫英文先抓「表」與「鍵」：relation/table, tuple/row, attribute/column。看到 ACID 就直接對應四個單字。

  • Normalization 是「去重複、避免異常」。
  • Primary key 唯一識別，foreign key 用來關聯。
  • SQL 題目先看 WHERE 條件。

Learning Objectives

  • Explain the relational model and keys.
  • Describe normalization goals and forms.
  • Write basic SQL queries.
  • Understand ACID properties and transactions.

Key Terms

relation, tuple, primary key, foreign key, normalization, ACID, SQL

Worked Example: Primary Key

In a Students(id, name, major) table, id is the primary key because it uniquely identifies each row.

Relational Model

Data is stored in tables (relations). Each row is a record (tuple), each column is an attribute.

Primary key uniquely identifies a record; foreign key links to another table.

Normalization

Normalization reduces redundancy and update anomalies. Common forms: 1NF, 2NF, 3NF.

SQL Basics

SQL supports SELECT, INSERT, UPDATE, DELETE. Example:

SELECT name FROM students WHERE major='CS';

Transactions & ACID

  • Atomicity
  • Consistency
  • Isolation
  • Durability
