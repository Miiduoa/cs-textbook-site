English Reading Guide (中文導讀)

ADT 的英文重點是 behavior（行為）與 interface（介面）。先抓「能做什麼」，再抓「如何實作」。

  • Abstract = 不談細節，只談規則。
  • 讀到 invariant 就是「一定要成立的條件」。
  • Set ADT 的 invariant：不重複。

Learning Objectives

  • Define an ADT and its interface/operations.
  • Distinguish ADTs from concrete data structures.
  • Describe common ADTs (set, map, queue, list).
  • Explain invariants and abstraction barriers.

Key Terms

ADT, interface, invariant, set, map, queue

Worked Example: Set Invariant

If insert(x) is called and x already exists, the set remains unchanged. This preserves uniqueness.

What is an ADT?

An Abstract Data Type (ADT) specifies behavior (operations and rules) without specifying the underlying implementation.

Example: A Stack ADT defines push, pop, and peek regardless of whether it is implemented using an array or linked list.

ADT vs Data Structure

ConceptFocusExample
ADTOperations + rulesSet, Map, Stack
Data StructureConcrete storageArray, Linked List, Hash Table

Common ADTs

  • List: ordered collection
  • Set: unique elements only
  • Map: key → value pairs
  • Queue: FIFO access
