Time & Space Complexity

We analyze how running time grows as input size n increases. Big‑O describes the upper bound of growth.

NotationNameExample
O(1)ConstantArray access
O(log n)LogarithmicBinary search
O(n)LinearLinear scan
O(n log n)LinearithmicMerge sort
O(n²)QuadraticNested loops

Growth Comparison

Interactive: Growth Estimator

O(n)=1000, O(n log n)=9966, O(n²)=1,000,000

Best vs Worst Case

Some algorithms perform very differently depending on input. Example: quicksort has average O(n log n) but worst O(n²) when partitions are poor.

Amdahl’s Law

Speedup is limited by the portion of a program that cannot be parallelized.

Speedup = 1 / ( (1−p) + p/s ) where p is parallel fraction and s is speedup of that part.
