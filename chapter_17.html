English Reading Guide (中文導讀)

資安英文常出現 CIA、symmetric/asymmetric、hash。讀到「攻擊」先找類型（SQLi/XSS/DoS），讀到「防禦」先找方法（validation/patching/least privilege）。

  • Confidentiality/Integrity/Availability = CIA 三要素。
  • Symmetric 快、Asymmetric 方便做金鑰交換。
  • SQL Injection 題目先看拼接點。

Learning Objectives

  • Explain CIA triad and common threats.
  • Compare symmetric and asymmetric cryptography.
  • Describe basic defenses (validation, hashing, MFA).
  • Estimate password strength with entropy.

Key Terms

CIA triad, encryption, hash, SQL injection, phishing, entropy

Worked Example: Symmetric vs Asymmetric

Symmetric uses one shared key (fast). Asymmetric uses public/private keys (enables key exchange and signatures).

CIA Triad

  • Confidentiality: prevent unauthorized access.
  • Integrity: prevent unauthorized modification.
  • Availability: ensure services remain usable.

Cryptography

TypeKeyExample
SymmetricSame keyAES
AsymmetricPublic/PrivateRSA, ECC
HashOne‑waySHA‑256

Common Threats

  • Phishing & social engineering
  • Malware (virus, ransomware)
  • SQL injection & XSS
  • Denial of Service (DoS)
