Software Development Life Cycle

Software engineering applies systematic processes to build reliable systems. Common models:

  • Waterfall: linear stages (requirements → design → implementation → testing → deployment).
  • Iterative: build in increments, revisit earlier stages.
  • Agile: short sprints, continuous feedback, user stories.

Requirements

Requirements define what a system should do:

  • Functional: features and behaviors (login, search).
  • Non‑functional: performance, security, usability, scalability.

Design & Architecture

Design turns requirements into structure. Key ideas:

  • Modularity & separation of concerns
  • Layered architecture (UI → Service → Data)
  • Design patterns (Singleton, Factory, Observer)

Testing

Testing improves quality and reduces defects:

LevelPurpose
UnitIndividual functions/classes
IntegrationInteractions between modules
SystemEnd‑to‑end behavior
AcceptanceUser validation

Version Control & CI/CD

Modern teams use Git for collaboration and CI/CD pipelines for automated builds, tests, and deployments.

Benefits: faster feedback, fewer integration bugs, consistent releases.
