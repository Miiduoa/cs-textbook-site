English Reading Guide (中文導讀)

這章英文重點在「結構 vs 操作」：insert, delete, search, traversal。先記操作再記適合的結構。

  • Traversal 是「走訪」，常見 preorder/inorder/postorder。
  • Adjacency matrix vs list 是圖的兩種表示法。
  • 看到 balanced 先聯想到 O(log n)。

Learning Objectives

  • Choose appropriate structures for different operations.
  • Explain arrays, lists, stacks, queues, trees, and graphs.
  • Compare time complexity trade‑offs.
  • Traverse a binary search tree.

Key Terms

array, linked list, stack, queue, tree, graph, traversal

Worked Example: BST Inorder Traversal

Insert 5, 9, 2, 7, 8, 0, 4, 1, 3, 6 into a BST and traverse inorder to get the sorted sequence.

Why Data Structures?

Data structures organize data so that operations (search, insert, delete) are efficient. The choice affects performance and memory usage.

Linear Structures

  • Array: fast indexed access, fixed size.
  • Linked list: dynamic size, slower random access.
  • Stack (LIFO): push/pop at top.
  • Queue (FIFO): enqueue/dequeue.

Trees & Graphs

Tree is hierarchical (binary search trees, heaps). Graph models networks with vertices and edges.
