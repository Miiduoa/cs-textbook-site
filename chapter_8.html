English Reading Guide (中文導讀)

這章的英文偏「方法論」：correctness（正確性）、optimal（最優）、strategy（策略）。看到演算法名稱時先記「想法」，再記「步驟」。

  • Greedy 是「每一步選最好」，不等於全局最好。
  • Dynamic programming 是「重用子問題解」。
  • 排序/搜尋題目先判斷資料是否「已排序」。

Learning Objectives

  • Describe algorithm properties: correctness, termination, efficiency.
  • Compare common design paradigms (greedy, divide & conquer, DP).
  • Analyze searching and sorting procedures.
  • Trace a recursive algorithm and identify base cases.

Key Terms

algorithm, correctness, greedy, divide and conquer, dynamic programming, recursion, base case

Worked Example: Selection Sort Trace

Sort [5,4,3,2,1] by repeatedly selecting the minimum and swapping into position.

Pass 1 → [1,4,3,2,5], Pass 2 → [1,2,3,4,5] (continue until sorted).

What is an Algorithm?

An algorithm is a finite set of steps that solves a problem. Good algorithms are correct, efficient, and easy to understand.

Algorithm Design Techniques

  • Brute force: try all possibilities.
  • Divide & Conquer: split, solve, combine (e.g., merge sort).
  • Greedy: locally optimal choices (e.g., Dijkstra).
  • Dynamic Programming: reuse subproblem results.

Searching

Linear search checks elements one by one. Binary search repeatedly halves a sorted list.

Sorting

Sorting organizes data to make searching and analysis easier.

AlgorithmAverageNotes
Bubble SortO(n²)Simple, slow
Merge SortO(n log n)Stable, uses extra memory
Quick SortO(n log n)Fast in practice

Recursion

Recursion solves a problem by calling itself on smaller inputs. Requires a base case to stop.

Example: factorial(n) = n × factorial(n−1)
